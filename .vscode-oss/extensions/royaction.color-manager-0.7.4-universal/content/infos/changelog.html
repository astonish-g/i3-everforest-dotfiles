v0.7.4

  • Fix: Hex6 to Hex3 conversion is fixed now
  • Fix: alpha slider in color picker converts alpha 0 for Hex colors to "00" now (was sometimes converted to "ff" before)
  • Fix: recognition of CSS-Colors when the cursor is placed on a CSS-Color value in the document (f.e. "PowderBlue" was recognized as a "Blue" before)
  • Fix: editing multiple document colors with "0x"-prefix are now correctly refreshed in the Color Picker (before the Picker had to be closed and reopened)

v0.7.3

  • The extension now opens in the sidebar
  • Commands are now categorized (open command palette and type "color manager")
  • New Function: Create Alpha-Steps
  • New Function: Create complementary Color
  • Handling: Sorting is now available in all modes by holding down the Alt-Key
    ‚ö†Ô∏è Attention! You may have to change a vscode setting to use this feature properly! Run command "color manager help" and read "Sort colors" for instructions!
  • Handling: Convert-, Sort- and Randomize-Function were moved to the extension-contextmenu and can be applied to selections
  • Settings: New setting "color-manager.contextMenu" -> Configure which commands appear in vscode's contextmenu
  • Settings: New setting "color-manager.paletteLimit" -> Palettes are now limited to 5000 entries by default. You can override the limit with this setting.
  • Settings: New setting "color-manager.fullWidth" -> Replaces the old setting "width" as the UI can now be resized directly with two bars at the left and the right side
  • Settings: New setting "color-manager.autoStartIntellisense" -> the current intellisense palette will be loaded automatically when vscode is restarted
  • Color Picker: add- and replace-Button are now permanently available at the same time
  • Color Picker: as long as the color picker is open, clicking on the color-fields in the palette will not insert the colors into your active document.
  • Color Picker: the color can be resetted by clicking on the right color preview field
  • Color Picker: when the picker is dragged, the position can be reseted with a doubleclick on the titlebar
  • Intellisense: the intellisense palette can now be chosen from a vscode quickpick (opening the user interface isn't required anymore)
  • Palettes: 3 new factory palettes added, "Affinity Photo Default Palette", "Jam3 Nice color palettes" with 500 color sets (2500 colors) and "Pantone C". New version of "Google Material Palette 2014" added (better arranged for the compact view mode). Run command "restore factory palettes" to get the new palettes (‚ö†Ô∏è Attention: overwrites existing factory palettes)
  • Palettes: The Palette Manager opens in full screen now
  • Fix: drag picker when window is scrolled
  • Fix: contextmenu behavior when multiple entries are selected
  • Fix: command "find colors in selection" should find all values now
  • Fix: higher precsision when converting hex to hsl and back to hex
  • ... heavy code refactoring ... no new bugs please üôè üòì
